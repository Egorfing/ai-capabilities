# AI Capabilities v0.2.1

Patch release to ship fixes after the 0.2.0 developer preview.

**Status:** Developer Preview / Early adopter release (same as 0.2.0) — recommended if you picked up the previous tag.

## What’s included
- Version bump across the workspace packages and example app.
- Type-safety fix in `defineCapability` (build now succeeds under `tsc` without type compatibility errors).
- Release automation docs/scripts updated to point at 0.2.1.

## Key capabilities (unchanged from 0.2.0)
- Extraction pipeline (OpenAPI, React Query, router, forms) producing canonical & public manifests.
- Helper API + runtime registry with policy enforcement and frontend action support.
- CLI commands (`init`, `inspect`, `extract`, `doctor`, `serve`, `prompt`, etc.).
- Example React app + happy-path guide, security model, FAQ, capability chaining, external-agent docs.

Upgrade by pulling the new tag, reinstalling dependencies, and rerunning `npm run build` / `npx ai-capabilities doctor` to confirm readiness.
