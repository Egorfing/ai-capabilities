# AI Capabilities v0.2.0

Expose real application actions to AI agents without rewriting your stack.

**Status:** Developer Preview / Early adopter release — focused on internal teams building capability surfaces and giving DX feedback.

## What AI Capabilities already delivers
- Extraction pipeline (OpenAPI, React Query, router, form hooks) → canonical & public manifests.
- `defineCapability` helper + runtime registry with policy enforcement.
- Frontend/UI action support (navigation, modals) via handler context adapters.
- Policy + security layer (visibility, risk levels, confirmation policy) surfaced through doctor checks.
- CLI tooling (`init`, `inspect`, `extract`, `enrich`, `serve`, `pilot`, `doctor`, `prompt`) plus prompt templates for schema/policy completion.
- Example React app + happy-path guide + FAQ for copy/paste onboarding.

## What’s new in 0.2.0
- **Runnable example app** — `examples/react-app` now includes its own `package.json`, Vite config, and deterministic agent so `npm run dev` works out of the box.
- **Capability chaining guidance** — new docs show how agents execute multi-step plans (e.g., `projects.create` → `navigation.open-project-page`).
- **External agent integration** — dedicated documentation for publishing `/.well-known/ai-capabilities.json`, exposing `/execute`, and wiring OpenAI/Claude tool calls.
- **Security model clarity** — consolidated docs for visibility/risk/confirmation policies plus doctor maturity summary in CLI output.
- **README + onboarding polish** — reorganized landing page, quickstart, and demo scenario so new contributors grasp the value within seconds.

## Primary commands

```bash
npx ai-capabilities init      # scaffold config + src/ai-capabilities
npx ai-capabilities inspect   # summarize extractable actions
npx ai-capabilities extract   # build raw/canonical/public manifests
npx ai-capabilities doctor    # readiness + safety report with maturity status
npx ai-capabilities serve     # expose runtime + /.well-known endpoint
npx ai-capabilities prompt    # generate LLM prompts for schema/policy/allowlist
```
