# AI Capabilities v0.1.0

Turn your software into an AI-compatible system.

**Status:** Developer Preview — ready for targeted pilot runs, not general availability.

## What’s in this release

- Automated capability extraction pipeline (OpenAPI, React Query, router, form hooks) feeding canonical/public manifests.
- Policy-aware capability runtime with binding resolver and HTTP transport (health, capabilities, execute, traces, well-known).
- Model/tool adapters (OpenAI, Anthropic, mock/internal) and enrichment pipeline for semantic metadata.
- Pilot runner that validates compatibility, produces diagnostics, and emits machine-readable + human-readable pilot reports.
- Inspect CLI for instant manifest summaries, filters, and JSON exports.

## Primary commands

```bash
npm run extract -- --project <path> --config <config>
npm run enrich -- --input ./output/ai-capabilities.json --model mock
npm run inspect -- --project <path> --public
npm run serve -- --config <config> --port 4000
npm run pilot -- --project <path> --with-enrich
```

## Who should try 0.1.0

- Internal platform engineers preparing AI agent surfaces for existing React/TypeScript backends.
- Tooling teams evaluating agentic capabilities for private pilots.
- Early adopters ready to provide feedback on manifests, adapters, and runtime gaps.

## Known limitations

- No bundled auth, rate limiting, streaming, or multi-tenant support yet.
- Manual handler bindings required; no auto-binding for HTTP/service actions.
- Pilot runner assumes single-repo projects; multi-repo orchestration is TBD.
- Smoke install currently mirrors workspace node_modules due to offline environment; real publish requires standard npm install with registry access.
- Repository/homepage metadata still needs canonical URLs before public publish.
