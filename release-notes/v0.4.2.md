# v0.4.2 — 2026-03-15

This release lands the UX/DX improvements we have been iterating on since 0.4.0: stricter manifest handling, an honest capability lifecycle report, and a production-ready React example with a mock backend and assistant overlay. 0.4.1 only reintroduced the README to the npm tarball; 0.4.2 contains actual feature work.

## CLI & runtime
- Added `npx ai-capabilities status` to summarize every capability’s lifecycle (`discovered → scaffolded → authored → registered → wired → executable`) with honest `yes / no / unknown` values and concrete “next step” notes.
- All commands that rely on project scaffolding now run a preflight check. In interactive shells the CLI offers to run `init` immediately; in CI/non-interactive mode it exits with a clear message that lists the missing artifacts.
- Introduced `resolveManifestSources` / `loadManifest`, a reusable helper that understands runtime URLs, explicit public manifest URLs, and local files with predictable precedence and caching. Public vs. internal manifests can no longer be mixed accidentally—`.well-known/ai-capabilities.json` only serves an explicit public manifest, otherwise it stays empty and logs the warning you saw during development.
- Runtime logging now spells out whether public or internal manifests were loaded, from which source, and whether a fallback or cache was used.

## Documentation
- Reworked `docs/quickstart.md`, `docs/mixed-scenarios.md`, and the README onboarding sections so they walk a new team all the way from `npx ai-capabilities init` to the first executable capability, highlighting which steps are automatic vs. manual.
- Added `docs/status.md`, `docs/runtime.md`, and updates to the manifest/public-boundary docs to explain the new helper, public snapshot workflow, and how to keep `.well-known` safe by default.

## React example
- The `examples/react-app` project now ships a JSON Server backend (`npm run dev:api`), React Router v6 navigation (Projects list ↔ Project detail), and a shared chat overlay that talks to the runtime on every page.
- Added `.env`/`.env.example` pairs so you can point the example at any internal LLM endpoint (no credentials baked in). The chat UI falls back gracefully when env vars are missing.
- Todos per project can be created/toggled through the assistant, demonstrating both read and mutation capabilities end to end.

## Packaging
- `npm pack` continues to include `README.md`, LICENSE, and the prebuilt `dist/**` outputs so npmjs.com renders the docs correctly. Run `npm pack` locally before publishing if you want to double-check the bundle.
