# v0.4.0 — 2026-03-15

## Highlights
- **Swagger 2.0 extraction**: the OpenAPI extractor now understands `swagger.json|yaml`, normalizes body/query parameters, and reports a per-run summary (`processed/succeeded/failed`). Unsupported specs emit explicit errors but do not stop other specs from processing.
- **Safer scaffold layout**: new projects scaffold into `src/app-capabilities`. The CLI auto-detects legacy `src/ai-capabilities`, warns once per command, and doctor flags the migration until you rename.
- **Quickstart & docs overhaul**: guides walk teams from `init` all the way to the first executable capability, explain manual vs. automatic steps, and document the new directory structure plus runtime wiring expectations.
- **Public manifest & loader UX**: HTTP runtime now relies on explicit `output/ai-capabilities.public.json`, with `npx ai-capabilities manifest public` and the `loadManifest(...)` helper making local vs. remote source selection predictable.
- **Capability lifecycle clarity**: `npx ai-capabilities status` surfaces discovered → scaffolded → authored → registered → wired → executable states with honest `yes/no/unknown` flags and next-step notes.
- **Diagnostics upgrades**: Preflight/doctor highlight missing init steps, absent public manifests, unsafe fallbacks, and legacy directories so teams catch issues before publishing.

## Details
- Added Swagger-aware parsing (body parameters, `responses[*].schema`, parameter merging) plus fixtures/tests covering mixed specs.
- Introduced central capability-dir resolver + warnings, updated CLI commands (init/scaffold/auto-bind/status/doctor) and docs to prefer `src/app-capabilities` while honoring legacy paths.
- Rebuilt quickstart narrative, README links, and workflow docs to emphasize manual authoring, registry wiring, and smoke tests before claiming “executable”.
- Added manifest helper docs + CLI command references so teams know how to produce/publicize `ai-capabilities.public.json` safely.
- Implemented lifecycle/status command docs/examples and ensured doctor/status share the same terminology.
- Release tooling: version bumped to `0.4.0`; see README for scaffold + runtime expectations when upgrading from 0.3.x.
